What just happened?

1. Your FastAPI app connects to PostgreSQL using the password from .env
2. When you check if a number is prime, the API:
   - Generates a unique transaction ID
   - Calculates if the number is prime
   - Saves the result to PostgreSQL
   - Returns the answer as JSON
3. You can retrieve any check later using its transaction ID
4. All data persists in PostgreSQL (survives server restarts!)

- API = The restaurant (takes orders, serves food)
- Database = The kitchen (stores and retrieves data)
- Models = The recipes (how to structure data)
- Schemas = The menu (what customers can order)
